# v0.20.0 Beta 4 Catalog Resolution Validation

## Scope

Beta 4 remains read-only and does not change the portal manual reassignment
workflow. It corrects the failure patterns found by the 25-track beta 3 report:

- apostrophes are preserved in MusicBrainz search terms;
- remaster labels are release evidence, not recording variants;
- a Roon album is a weak hint and cannot force a live/compilation recording;
- a structured album artist can recover primary-artist intent from legacy
  tracks with flat performer/composer display credits;
- compatible duplicates advance only when one candidate has unique ISRC
  evidence, and remain medium confidence;
- release lookup paginates beyond the 25 embedded MusicBrainz entities;
- bracketed edition years such as `[2003]` do not become part of an album title;
- truncated release browsing remains insufficient evidence rather than a false
  catalog conflict;
- beta 3 recording-resolution cache entries are invalidated.

No MCP tool name, input schema or widget resource changed. ChatGPT Actions do
not need to be updated for this beta.
