# v0.9.2 Validation

v0.9.2 fixes widget hydration and false-positive playback confirmations.

## Observed Problems

- ChatGPT mounted the Roon Control widget, but it remained at
  `Waiting for a Roon action...`.
- The model reported that Despacho was playing even though the zone remained
  paused.

## Fixes

- Widget URI changed to `ui://roon-ai-bridge/control-v3.html` to invalidate
  cached HTML.
- The widget reads initial `window.openai.toolInput` and
  `window.openai.toolOutput`.
- The widget listens for `openai:set_globals` and MCP Apps bridge
  `ui/notifications/*` updates.
- Playback commands return `zone_name`, `previous_state`, `state` and
  `state_verified`.
- Play, pause, playpause and stop fail with `PLAYBACK_STATE_NOT_CHANGED` if
  Roon does not reach the expected state within eight seconds.

## Automated Tests

```bash
pnpm run test
```

Validated locally:

- Widget v3 serves the MCP Apps MIME type.
- Widget v3 supports initial ChatGPT hydration and MCP result notifications.
- A paused zone changed to playing and returned `state_verified: true`.
- The complete suite passes.

## Live MCP Validation

Validated against Despacho through the deployed `/mcp` endpoint:

- `pause` changed the zone from `playing` to `paused`.
- `play` changed the zone from `paused` to `playing`.
- Both results identified `zone_name: "Despacho"` and returned
  `state_verified: true`.
- `resources/list` exposes `ui://roon-ai-bridge/control-v3.html`.
- `roon_control_playback` points to the same widget v3 URI.

The final state was left as `playing`.
