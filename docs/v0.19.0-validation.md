# v0.19.0 Validation

## Scope

This validation covers the local implementation of:

- GitHub Actions validation and multi-architecture GHCR publication;
- image-based LXC and general Docker installation;
- portal-driven image updates with backup, health verification and rollback;
- PolyForm Noncommercial licensing and the commercial-use policy;
- transactional, recorded SQLite migrations and duplicate-schema removal;
- UTF-8 Spanish text correction and regression scanning; and
- graceful SIGTERM/SIGINT shutdown.

## Publication And Live Validation Still Required

No commit, push, GitHub publication or LXC deployment was performed as part of
this local implementation. After the changes are pushed:

1. Confirm the `Publish container image` workflow succeeds for `beta`.
2. Make the GHCR package public once in its GitHub package settings.
3. Confirm both amd64 and arm64 manifests exist for the `beta` image.
4. From the current LXC portal, request the beta update and confirm it moves
   through download, backup, restart and verification.
5. Confirm `data/installed-release.json` matches the published commit and image
   digest, and `/roon/status` reports the core, transport and browse as ready.
6. Confirm playlists, settings, portal access and Roon authorization survive the
   migration.
7. Perform one controlled rollback drill with no audible playback: use an
   unavailable test image tag and verify that the previous container and data
   return automatically.
8. Refresh the ChatGPT app and start a new conversation only if deployed MCP
   metadata or widget resources changed.

## Expected Update Behavior

A source push does not immediately become installable. RoonIA offers only the
newest commit whose image workflow completed successfully. Therefore the usual
developer loop remains: commit, push, wait for the GitHub check, and press the
portal update button. No local or LXC compilation is required.
