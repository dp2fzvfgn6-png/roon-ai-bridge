# v0.19.0 Validation

## Scope

This validation covers the local implementation of:

- GitHub Actions validation and multi-architecture GHCR publication;
- image-based LXC and general Docker installation;
- portal-driven image updates with backup, health verification and rollback;
- PolyForm Noncommercial licensing and the commercial-use policy;
- transactional, recorded SQLite migrations and duplicate-schema removal;
- UTF-8 Spanish text correction and regression scanning; and
- graceful SIGTERM/SIGINT shutdown.

## Publication And Live Validation

The beta application commit `824459e3b63da96afdea5c92b5581c1b0b8725c8`
passed the GitHub Actions test/build gate and published an anonymous, publicly
pullable OCI index containing both `linux/amd64` and `linux/arm64` images.

The existing v0.18.1 beta LXC was then upgraded through the new image updater.
The migration created a 5.0 MiB pre-update backup and completed with:

- container state `running/healthy`;
- installed version `0.19.0`, beta channel and the published Git revision;
- matching checkout, image label, image digest and `installed-release.json`;
- active and enabled systemd update watcher;
- database, Roon Core, MCP catalog and migrations all ready;
- Roon Core, transport, Browse and image services connected;
- MCP initialization reporting `roon-ai-bridge` v0.19.0;
- 42 MCP tools, all required intents and six widgets present; and
- no recent application errors after startup.

The backup/rollback implementation is covered by the automated and script
contract checks. An intentional live failure drill was not performed against
the working Roon installation because the successful migration already retained
the previous image and backup as recovery points.

## Expected Update Behavior

A source push does not immediately become installable. RoonIA offers only the
newest commit whose image workflow completed successfully. Therefore the usual
developer loop remains: commit, push, wait for the GitHub check, and press the
portal update button. No local or LXC compilation is required.
