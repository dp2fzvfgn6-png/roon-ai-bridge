# v0.17.2 Playlist Cover File Validation

## Scope

This validation covers reliable generated playlist artwork handoff from
ChatGPT to RoonIA and the stored image quality policy.

## Model flow

1. `roon_prepare_playlist_cover` is called before image generation with an
   exact playlist ID or name.
2. The result includes playlist metadata, up to 24 representative tracks,
   1024×1024 generation guidance, the 768×768 minimum and ordered next steps.
3. ChatGPT generates one square image from that context.
4. `roon_set_playlist_cover` receives the generated image in `image_file`.
5. RoonIA downloads, validates, normalizes, stores and reports verified output
   metadata before the model claims success.

The MCP server instructions keep this sequencing rule in their first 512
characters. The upload tool declares `_meta["openai/fileParams"]` with the
top-level `image_file` schema required by ChatGPT Apps. Inline data URLs and raw
Base64 remain supported for older clients.

## Download safety

- Only public HTTPS URLs are accepted.
- Loopback, link-local and private literal IP addresses are rejected.
- Downloads time out after 15 seconds and stop once they exceed 5 MB.
- JPEG, PNG and WebP are the only accepted MIME types.
- The temporary `download_url` is redacted from technical and action logs.
- Existing byte-signature validation still rejects MIME/content mismatches.

## Image quality

- Sources below 768 pixels on either oriented axis are rejected with the
  received and required dimensions.
- Accepted sources are auto-rotated and center-cropped to a square.
- Output is a metadata-free 1024×1024 WebP.
- Compression begins at quality 88 and steps down only as needed to remain at
  or below 750 KB.
- There is no 512×512 fallback; RoonIA fails explicitly rather than silently
  lowering stored resolution.
- Successful tool results include stored width, height, format, color space and
  byte count.

## Automated coverage

Focused tests cover:

- preflight playlist resolution and generation context;
- the exact ChatGPT file-parameter manifest shape;
- authorized file download, MIME validation, byte limits and private URL
  rejection;
- redaction of temporary download URLs;
- legacy Base64 compatibility;
- rejection of 256×256 inputs;
- square 1024×1024 normalization and stored metadata verification.

Run the complete release checks with:

```powershell
pnpm run test
pnpm run build
```

No deployment or live ChatGPT validation is implied by this local validation
document.
