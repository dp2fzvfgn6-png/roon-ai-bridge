# v0.18.0 Architecture Validation

## Scope

This release reorganizes the project without intentionally changing public HTTP,
portal, MCP, Roon or persistence behavior. It removes inactive code and
introduces smaller internal boundaries behind the existing facades.

## Implemented Boundaries

- `src/app` owns the typed shared context and dependency composition.
- `src/bridge-v2/mcp/index.ts` is the only stdio MCP entry point; the
  disconnected legacy `src/mcp` tree is removed.
- `src/portal/routes` separates authentication, dashboard, connection, system,
  access and audio-administration endpoints.
- `portal/features/mini-player.js` owns the existing mini-player browser logic.
- `src/roon/media` owns media contracts and pure search policy.
- `src/services/playlists` owns playlist contracts and cover policy.
- `src/bridge-v2/intents/transportIntentHandler.ts` owns state and transport
  intentions behind `IntentGateway`.

## Preserved Contracts

- HTTP paths, payloads, authentication and homogeneous error responses.
- Portal routes, markup, visual behavior and update workflow.
- MCP names, schemas, annotations, result envelopes and widget resource URIs.
- Roon command verification, ambiguity policy and safe-volume behavior.
- SQLite schema, playlist/track identity and persisted data formats.
- Single-process sharing of the Roon client and application services.

## Manual Validation After Deployment

1. Confirm the deployed commit and reported version are the intended v0.18.0
   release, and `/roon/status` reports core, transport and browse ready.
2. Log into the portal and visit Inicio, Música, Playlists, Conexiones, Sistema
   and Actualización. Confirm navigation and data load without console errors.
3. Select Despacho in the mini-player. Inspect its current volume before any
   audible action; do not raise it for the test. Exercise play/pause, next,
   seek and volume at the existing safe level, then restore the original state.
4. Open an existing saved playlist and verify its name, cover, ordered tracks
   and playback target. If testing mutations, use a disposable playlist and
   remove only that playlist after confirming the destructive action.
5. Refresh the ChatGPT app connection and start a new conversation. Verify
   status/zones, now playing, media search, queue and saved-playlist views, then
   pause and resume Despacho through the corresponding canonical MCP tools.
6. Verify an ambiguous media query returns choices without starting playback.
7. Review application logs for unexpected errors and confirm the final zone,
   grouping, volume, playback and queue state are the intended ones.
