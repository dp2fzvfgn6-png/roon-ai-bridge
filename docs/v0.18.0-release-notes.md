# RoonIA v0.18.0 Release Notes

Date: 2026-07-20
Release channel: stable (`main`)
Source channel: beta

## Summary

v0.18.0 promotes the architecture-focused beta that followed v0.17.3. It
reorganizes the application into smaller typed modules while preserving the
existing HTTP, portal, MCP, Roon and persistence contracts.

## Architecture changes

- A typed composition root now constructs the shared API, portal and MCP
  dependencies without post-construction context mutation.
- The disconnected legacy MCP implementation has been removed. `bridge-v2` is
  now the only HTTP and stdio MCP boundary.
- Portal administration routes are split into focused backend modules, and the
  persistent mini-player is extracted as a browser feature.
- Media, playlist and transport policies now live behind smaller internal
  boundaries while their public service and intent facades remain compatible.
- Architecture documentation describes the resulting structure and the
  remaining extraction candidates.

## Compatibility

The release intentionally retains the existing HTTP paths, authentication,
portal presentation, MCP tool names and schemas, widget v19 resource URIs,
Roon safety behavior, SQLite schema and persisted data formats.

After deployment, refresh the ChatGPT app and start a new conversation so it
reloads tool metadata from the canonical MCP v2 implementation.

## Upgrade and rollback

Stable installations follow `main`; installations with beta updates enabled
follow `beta`. The LXC updater preserves runtime data and configuration while
rebuilding from the selected Git ref.

Before rollout, back up `/opt/roon-ai-bridge/data` and the LXC `.env`. Rollback
should select the previous validated tag and use the documented LXC update
procedure.

See [Update Existing LXC](update-lxc.md),

[v0.18.0 Architecture Validation](v0.18.0-architecture-validation.md) and the
