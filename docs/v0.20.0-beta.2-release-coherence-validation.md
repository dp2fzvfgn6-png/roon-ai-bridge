# v0.20.0 Beta 2 Release Coherence Validation

## Scope

Beta 2 extends the read-only identity V2 diagnostic introduced in beta 1. It
does not update a playlist. Its purpose is to prove that recording metadata and
release metadata can be kept coherent before the write path is changed.

The deliverable adds:

- separate MusicBrainz recording, release-group and release identities;
- explicit release evidence from an LLM/user album hint or a lower-confidence
  Roon album observation;
- exact edition selection only when a single release remains;
- release-track duration and position verified inside that exact release;
- recording median duration when no exact edition can be selected;
- Cover Art Archive URLs bound to a release or release group;
- explicit warnings when the existing Roon image key cannot be compared with
  the catalog artwork;
- a v2 provider cache key so beta 1 responses cannot hide the new release
  fields for seven days.

The portal manual search, reassignment and playlist metadata refresh paths are
unchanged.

## Resolution contract

The release resolver uses this order:

1. MusicBrainz must first resolve the requested recording.
2. An explicit `llm_hints.album` is the strongest release-title anchor.
3. A stored Roon album may be inspected as an observation, but is never treated
   as authoritative intent.
4. If the title points to several release groups, the result is
   `ambiguous_release`.
5. If one release group remains but several editions remain, the result is
   `release_group_candidate`.
6. A unique release, or a unique release matching an explicit year hint, is
   looked up directly. The recording must occur exactly once in that release.
7. Only that last case returns `exact_release`, edition track position and an
   edition track duration.

MusicBrainz defines recording length as the median length of its associated
release tracks. Consequently, a recording-level duration is returned with
`source=musicbrainz_recording_median` and `exact_for_release=false`. An exact
edition track uses `source=musicbrainz_release_track`.

## Direct provider validation

The implementation was checked against the public MusicBrainz JSON API on
2026-07-22 using `Purple Haze` by The Jimi Hendrix Experience:

- recording MBID `107af95b-776e-40af-a677-328ce4a72f16`;
- release MBID `84eba72b-cb7e-4771-bd05-9fdf2daccd35`;
- release-group MBID `da40e720-2097-3730-8a7f-0c2ddbff4a96`;
- medium 1, track 1 (`A1`), track length 173240 ms;
- the exact release declares front artwork in Cover Art Archive.

The test also established that `releases+media` is insufficient for nested
release-group metadata. `release-groups` must be included explicitly in the
recording lookup.

## Automated validation

Run:

```powershell
pnpm run test
pnpm run build
```

Focused tests cover:

- nested release MBIDs, release-group MBIDs and track offsets;
- exact release-track duration and position;
- multiple editions stopping at a release group;
- an explicit year selecting one edition;
- a wrong Roon album being reported rather than replaced by a popular album;
- read-only MCP delivery.

## Post-deployment user validation

After deployment, refresh the ChatGPT app and start a new conversation so the
updated MCP description is loaded.

1. Call `roon_get_playlist` for `rock-clasico-esencial` and retain the before
   state for `Purple Haze`, `Born To Be Wild` and `Won't Get Fooled Again`.
2. Call `roon_analyze_playlist` with those three `catalog_track_ids`.
3. Confirm `mode=shadow`, `mutates_playlist=false` and
   `identity_contract_version=2`.
4. Inspect each track's `release_result`:
   - an exact release must contain a release MBID and
     `duration.source=musicbrainz_release_track`;
   - a group-only result must use `release_group_candidate`, keep
     `release=null` and label duration as `musicbrainz_recording_median`;
   - an observed album that is not present for the recording must produce
     `not_found`, not a substituted greatest-hits album.
5. If `cover_art` is present, open `front_500_url`. Its `entity` and MBID must
   match the exact release or release group in the same result.
6. Confirm a Roon image key remains `cover_coherence=unverified`; beta 2 must
   not claim the images are equal merely because album titles match.
7. Call `roon_get_playlist` again and verify the stored identity, album,
   duration, artwork and selected Roon result are unchanged.
8. Repeat the diagnostic and verify the second request uses the persistent
   provider cache.

Do not use the portal `Actualizar metadatos` action to validate beta 2. It still
uses the existing v0.19 write path intentionally.

## Exit criteria for the next deliverable

Do not connect identity V2 to playlist writes until real beta diagnostics show:

- no composer is promoted to primary artist;
- no ambiguous release is silently selected;
- every exact duration is backed by an exact release track;
- every proposed catalog artwork URL belongs to the selected entity;
- existing manual Roon selections remain locked and unchanged.
