## v0.5 Target

This validates the v0.5 target:

- Keep the v0.1 transport and volume control features working.
- Keep the v0.2 library browse features working.
- Keep the v0.3 search and play-by-query features working.
- Keep the v0.4 queue management features working.
- Add local virtual playlists stored in `data/virtual-playlists.json`.
- Support creating, listing, reading and deleting virtual playlists.
- Support adding and removing virtual playlist tracks.
- Support playing a virtual playlist with `add_to_queue`, `add_next` and `play_now`.
- Keep direct TIDAL write access, OpenAI, MCP, Cloudflare and auth out of scope.

One fix was made while validating against the live Roon Core:

Before that fix, Roon did start playback correctly, but the HTTP response reported a false failure because the bridge kept browsing after the action had already been executed.

## Notes

Virtual playlist tracks store Roon search queries, not immutable Roon track IDs. This keeps v0.5 simple and works well for explicit track queries such as `bad bunny dakiti`, but future phases may add richer track resolution and saved Roon metadata.

Repeated validation calls add repeated tracks to the live Roon queue. This is expected for queue action tests.

## Status
