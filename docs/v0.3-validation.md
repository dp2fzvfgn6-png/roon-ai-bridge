# v0.3 Validation

Date: 2026-06-24

Environment:

- Dedicated Proxmox LXC: `roon-ai-bridge`
- Roon Core: separate VM/LXC on the same VLAN/subnet
- Docker Compose with `network_mode: host`
- `ENABLE_BROWSE=true`

## v0.3 Target

This validates the v0.3 target:

- Keep the v0.1 transport and volume control features working.
- Keep the v0.2 library browse features working.
- Expose Roon browse search through `GET /roon/search?q=...`.
- Start playback from a simple query through `POST /roon/play`.
- Keep OpenAI, MCP, auth, playlists and queue management out of scope.

## Validated Results

The v0.3 search/play-by-query flow was tested against the live Roon Core and worked.

Validated as working:

- Search endpoint responds through Roon browse search.
- Play-by-query endpoint starts playback in a selected zone.
- Search can use Roon-connected services exposed by Roon, such as TIDAL, without using any direct TIDAL API.

## Notes

v0.3 does not rank results intelligently. It uses the first plausible playable result and follows Roon browse actions until playback starts or no playback action is found.

## Status

v0.3 is validated as working for basic search and play-by-query.
