# v0.9 Validation

v0.9 was validated against the deployed Roon Core and the production LXC.

## Automated Tests

```bash
pnpm run test
```

Validated:

- TIDAL and quality metadata classification.
- Configured streaming catalog classification.
- Artist catalog and artist radio select different Roon actions.
- Media relevance is ranked before source preference.
- OAuth tokens are bound to resource and scope.
- PKCE methods other than `S256` are rejected.

## Typed Search

```bash
curl -G http://localhost:3000/roon/media/search \
  -H "Authorization: Bearer <API_TOKEN>" \
  --data-urlencode "q=Bad Bunny" \
  --data-urlencode "types=track,album,artist" \
  --data-urlencode "zone_id=<ZONE_ID>" \
  --data-urlencode "count=5"
```

Validated results:

- Tracks, albums and artists are returned separately.
- Roon playlists are returned as typed playlist results.
- Every result has a temporary `result_id`.
- A search for Bad Bunny ranks releases by Bad Bunny ahead of an unrelated album
  named `Bad Bunny`, while still using TIDAL as a tiebreaker for equally relevant
  results.
- Missing source or quality data is reported as `unknown`, not guessed.

## Playback

Validated in zone `16017884276cc06e0e4c92d966765d65ec86`:

- Track `DtMF` replaced the previous queue and left one queue item.
- Album `Un Verano Sin Ti` replaced the queue with 23 tracks and started `Moscow Mule`.
- Artist-only playback for Daft Punk used Roon `Shuffle`, started `Around the World`
  and returned only Daft Punk in the resulting queue.
- Artist radio seeded with Daft Punk started similar-music playback.
- Appending `DtMF` preserved current playback and placed it last.
- Playlist `Mix Tidal DUB` was identified as a TIDAL playlist, replaced the queue
  with its 40 tracks and started `I Wonder`.

## MCP And OAuth

Validated:

- MCP initialization reports `roon-ai-bridge` version `0.9.0`.
- Tools expose `outputSchema`.
- Widget URI is `ui://roon-ai-bridge/control-v2.html`.
- OAuth metadata advertises only PKCE `S256`.
- New tools include typed search, media details, artist releases, exact playback, artist radio and exact queue addition.
- Legacy query tools have `visibility: ["app"]`; typed media tools have
  `visibility: ["model", "app"]`.

## Known Limitations

- Roon search rows often omit explicit service and audio-quality metadata.
- Source classification uses explicit labels first and configured linked-catalog markers second.
- Release rows may omit dates, so latest/earliest ordering cannot yet be guaranteed.
- The Roon Extension API cannot remove, reorder or clear individual queue items.
