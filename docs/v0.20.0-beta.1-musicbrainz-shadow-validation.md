# v0.20.0 Beta 1 MusicBrainz Shadow Validation

## Scope

This first v0.20.0 deliverable introduces the infrastructure needed to replace
role-blind Roon display credits without changing any playlist identity:

- additive identity V2 shadow contract;
- persistent SQLite cache for MusicBrainz recording resolutions;
- serialized requests and bounded retry for MusicBrainz HTTP 429/503 responses;
- selected-track MusicBrainz diagnostics through `roon_analyze_playlist`;
- no writes to playlist tracks, Roon bindings or catalog metadata;
- no changes to the portal manual search and reassignment workflow.

The beta deliberately does not use identity V2 to create, repair or refresh a
playlist yet. A diagnostic may therefore expose a wrong legacy observation,
but it cannot persist or act on it.

## Identity V2 shadow contract

Each selected diagnostic track returns:

- `intent`: requested title, primary/featured artists, album/year hints and
  recording intent;
- `recording`: MusicBrainz recording identity when uniquely resolved;
- `release`: the current Roon release observation, still without a MusicBrainz
  release or release-group MBID in beta.1;
- `credits`: primary/featured proposal credits, MusicBrainz composer/lyricist
  relations and role-unknown Roon display credits in separate collections;
- `roon_selection`: the existing result/binding and whether an explicit
  selection is locked;
- `evidence`: the source used by each identity component.

`shadow: true` and `mutates_playlist: false` are invariants for this release.
A unique but unanchored MusicBrainz result is reported as `candidate_recording`;
only a result with strong album/ISRC evidence may be `exact_recording`.

## Local automated validation

```powershell
pnpm run test
pnpm run build
```

Focused coverage verifies:

- additive migration `003_metadata_provider_cache`;
- cache reuse after recreating the MusicBrainz service;
- status-aware expiry and stale-entry cleanup;
- bounded 503 retry;
- preservation of `Steppenwolf` as primary artist while `Mars Bonfire` remains
  a MusicBrainz composer and an unclassified Roon display credit;
- read-only playlist diagnostics;
- MCP gateway delivery of the shadow result.

## Post-deployment user validation

After deploying the beta image, refresh the ChatGPT app and start a new
conversation so the updated `roon_analyze_playlist` schema is loaded.

1. Call `roon_get_playlist` for `rock-clasico-esencial` and retain the complete
   before-state for three tracks, including `track_id`, `identity`,
   `selected_result_id`, album and duration.
2. Call `roon_analyze_playlist` with `catalog_track_ids` for:
   `Purple Haze`, `Born To Be Wild` and `Won't Get Fooled Again`.
3. Confirm the response contains:
   `mode: "shadow"`, `mutates_playlist: false`,
   `identity_contract_version: 2` and three diagnostics.
4. Confirm `intent.primary_artists` contains the actual requested performer,
   not every Roon subtitle credit. For `Born To Be Wild`, `Steppenwolf` must be
   primary while `Mars Bonfire` may appear under `roon_unclassified` and, when
   MusicBrainz resolves the recording, under `composers`.
5. A provider ambiguity must return `ambiguous_recording`; it must not choose a
   duration or edition silently.
6. Call the same analysis a second time. It should complete substantially
   faster and `cache.active_entries` must be nonzero.
7. Restart the application container and repeat the same call. The persistent
   cache should still be used.
8. Call `roon_get_playlist` again and compare it with step 1. Track identity,
   Roon selection, album, artwork and duration must be unchanged.
9. In portal debug mode, open the existing manual reassignment search for one
   song, perform a search and cancel it. Its layout, candidates and behavior
   must be unchanged.

Do not use **Actualizar metadatos** to validate beta.1: that mutation still
uses the v0.19 enrichment pipeline and is intentionally outside this first
deliverable.

## Known limitations

- Diagnostics are limited to ten explicitly selected tracks per call.
- MusicBrainz recording identity is available; exact release, release group,
  medium and track MBIDs are scheduled for beta.2.
- Legacy tracks without proposal hints may recover the artist from their stored
  query. If that cannot be done without guessing, the diagnostic reports
  `insufficient_input`.
- Identity V2 is returned only in shadow diagnostics and is not persisted.
