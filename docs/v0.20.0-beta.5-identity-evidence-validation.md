# v0.20.0 Beta 5 Identity Evidence Validation

## Scope

Beta 5 remains a read-only identity V2 shadow diagnostic. It fixes the failures
found by the 25-track beta 4 report without changing playlist rows, Roon
selections, playback, MCP tool names, input schemas or widget resources.

The corrected invariants are:

- a cataloged ISRC is identity evidence only when it matches an observed ISRC;
- duplicate compatible recordings remain ambiguous without that match;
- a stored duration can constrain a recording only with `roon` or
  `musicbrainz_release_track` provenance;
- legacy stored queries can recover an exact edge-separated artist in either
  `title + artist` or `artist + title` order;
- a release anchor is searched by recording MBID and title before a bounded
  complete-catalog browse;
- observed release year, disc and track positions may narrow candidates, but
  exact release status still requires a unique MusicBrainz track verification;
- recording-resolution cache keys advance to v5 so beta 4 false positives are
  not reused.

## Automated validation

Run locally:

```powershell
pnpm run test
pnpm run build
```

Focused regression coverage confirms:

- an ISRC present on only one duplicate no longer selects that recording;
- an independently observed matching ISRC may select the corresponding
  recording;
- Paint It Black recovers The Rolling Stones from an artist-first legacy query;
- an unprovenanced legacy duration is excluded from the provider lookup;
- a targeted recording-MBID/release-title query is cached and can find an
  anchor beyond the embedded release list;
- Roon year, disc and track observations can select one edition, followed by a
  verified release-track lookup and exact duration provenance;
- negative or truncated release catalogs remain conflicts or insufficient
  evidence instead of becoming false exact results.

## Deployment validation

After an explicit beta deployment, refresh the ChatGPT app and start a new
conversation. Analyze `Rock clásico esencial` in batches of 10, 10 and 5.

Confirm:

1. Every batch reports `mode=shadow`, `mutates_playlist=false` and
   `identity_contract_version=2`.
2. All Along the Watchtower and Smoke on the Water are not exact recordings
   merely because one compatible result has cataloged ISRCs.
3. Paint It Black reconstructs The Rolling Stones when that artist appears at
   either edge of the stored query, while Mick Jagger and Keith Richards remain
   unclassified or work credits rather than inferred performers.
4. More Than a Feeling remains ambiguous unless the stored track supplies the
   same ISRC as the selected MusicBrainz recording.
5. House of the Rising Sun can find a positive album anchor through the
   targeted MBID/title query even when the complete browse would exceed 300
   releases. A negative truncated browse remains
   `release_catalog_truncated_before_anchor`.
6. Exact editions use matching observed year/disc/track evidence and report
   `duration.source=musicbrainz_release_track` with
   `duration.exact_for_release=true`.
7. Legacy stored-duration audit warnings may remain visible, but those values
   must not participate in recording matching without trusted provenance.
8. A repeated batch uses memory cache entries with zero provider requests; a
   restart uses persistent cache entries.
9. Playlist count, order, track IDs, `updated_at`, Roon bindings and playback
   state remain unchanged.
