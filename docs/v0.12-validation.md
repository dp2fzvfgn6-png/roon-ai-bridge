# v0.12 Validation

v0.12 expands RoonIA into a native-configurable Roon extension with advanced
transport, artwork, generic Browse flows, administrator accounts, zone presets
and safe host-supervised updates.

## Automated Validation

```bash
pnpm run test
pnpm run build
node --check portal/app.js
bash -n scripts/lxc-update-app.sh
```

Focused coverage includes:

- first administrator bootstrap, salted password hashes and hashed sessions;
- managed API-key role enforcement;
- safe port override persistence and fixed update requests;
- native seek, mute, pause-all, standby and playback-setting calls;
- generic Browse input, `replace_item` and `remove_item`;
- output subscription/cache integration;
- zone preset ordering, grouping verification and volume restoration;
- artwork-aware MCP widget v4;
- all earlier playback, playlist, OAuth, grouping and transfer behavior.

## Deliberate SDK Exclusions

- `stop_discovery()`, `disconnect_all()` and `ws_connect()` are not normal user
  actions and could strand the bridge.
- Provided `volume-control` and `source-control` services represent physical
  hardware. RoonIA controls existing Roon outputs and must not impersonate a
  device.

## Live Validation

Pending deployment checks:

- Roon shows Settings and Status for the extension.
- Roon settings show the LXC address and persist alternate ports.
- Check, restart and systemd-supervised update requests complete.
- A fresh portal asks for bootstrap token, username and password.
- Existing installations preserve API/OAuth data and playlists.
- `/roon/status` reports transport, Browse and Image readiness plus outputs.
- Seek, mute, standby, playback settings and global actions work on supported
  real outputs.
- Artwork loads in portal and ChatGPT widget v4.
- Generic Browse handles a real `input_prompt`, `replace_item` or `remove_item`
  when the Core exposes one.
- Zone presets group compatible outputs with the primary first and restore
  saved volumes safely.
