## v0.4 Target

This validates the v0.4 target:

- Keep the v0.1 transport and volume control features working.
- Keep the v0.2 library browse features working.
- Keep the v0.3 search and play-by-query features working.
- Expose queue snapshots through `GET /roon/queue/:zone_id`.
- Support `add_next` from a Roon search query.
- Support `add_to_queue` from a Roon search query.
- Support `play_from_here` for a queue item.
- Keep OpenAI, MCP, auth and virtual playlists out of scope.

## Validated Results

The LXC was updated from GitHub before testing.

`GET /roon/status`:

- `core_connected: true`
- `core_name: Roon Server`
- `transport_ready: true`
- `browse_ready: true`

`GET /roon/capabilities`:

- `queue: true`

`GET /roon/queue/16017884276cc06e0e4c92d966765d65ec86?max_item_count=10`:

- Returned queue snapshots for the `Despacho` zone.
- Returned Roon `queue_item_id` values.

`POST /roon/queue/:zone_id` with `inspect_actions`:

- Confirmed that a broad query such as `bad bunny` may expose only `Play Artist`.
- Confirmed that more specific track queries expose queue actions.

`POST /roon/queue/:zone_id` with `add_next`:

- Query: `bad bunny dakiti`
- Result: `DÁKITI - Bad Bunny / Jhayco` was added as next in the queue.
- Playback was not interrupted.

`POST /roon/queue/:zone_id` with `add_to_queue`:

- Query: `bad bunny monaco`
- Result: `MONACO - Bad Bunny` was added to the end of the queue.
- Playback was not interrupted.

Additional `add_to_queue` check:

- Query: `bad bunny neverita`
- Result: `Neverita - Bad Bunny` was added behind the current queue item.

`POST /roon/queue/:zone_id` with `play_from_here`:

- Queue item: `Neverita - Bad Bunny`
- Result: Roon jumped to `Neverita`.
- The endpoint returned `200 OK`.

## Fixes During Validation

Two fixes were made while validating against the live Roon Core:

- `cc198fe`: return success after executing Roon queue actions.
- `0be291a`: avoid waiting forever when `play_from_here` changes playback but Roon does not complete the callback promptly.

## Notes

Roon browse actions depend on the selected result. Broad artist queries can expose different actions than track or album queries. For predictable queue behavior, use specific song or album queries.

v0.4 is validated as working for basic queue management.
