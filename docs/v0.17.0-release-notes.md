# RoonIA v0.17.0 Release Notes

Date: 2026-07-13
Release channel: stable (`main`)
Source channel: beta

## Summary

v0.17.0 promotes the tested beta line that followed v0.16.0. It combines the
playlist identity work prepared as v0.16.1 with the new MCP v2 surface,
focused widgets, connection management, portal redesign and reliability fixes.

## User-visible changes

- The administration portal has a new responsive layout, persistent active
  zone selection, authenticated artwork and direct queue actions.
- ChatGPT receives three focused experiences: player, media explorer and
  library/playlist navigation.
- Search and playlist playback prefer the intended clean recording and expose
  ambiguity instead of silently choosing a close alternate version.
- ChatGPT OAuth clients, approval PIN rotation and generic MCP client profiles
  can be managed from the portal.
- Updates report their terminal state and distinguish builds even when the
  semantic version has not changed.

## MCP compatibility

The MCP v2 facade exposes 34 tools: 30 canonical intents, three model-visible
widget entry points and one app-only navigation tool. It does not expose the
legacy broad tool catalog or legacy aliases.

Tool descriptions begin with `Use this when...`, return concise structured
content and keep cross-tool sequencing in server instructions. Zone and output
arguments accept semantic names or stable identifiers.

The widget resources are:

- `ui://roon-ai-bridge/v11/player.html`
- `ui://roon-ai-bridge/v11/media-explorer.html`
- `ui://roon-ai-bridge/v11/library.html`

After deployment, refresh the ChatGPT app and start a new conversation so the
host reloads tool metadata and versioned widget resources.

## Playlist and playback safety

Virtual playlist tracks keep a permanent RoonIA `track_id`, recording snapshot
and identity fingerprint. Temporary `result_id` and `roon_item_key` values are
diagnostic observations only. Before playback, RoonIA searches again and
resolves a fresh action reference. If the first track is missing or ambiguous,
the current queue is left unchanged.

Live playback validation must follow the repository safety rules: inspect
Despacho first, never increase volume automatically and verify final queue and
playback state rather than trusting only a successful callback.

## Upgrade and rollback

Stable installations follow `main`; installations with beta updates enabled
follow `beta`. The LXC updater preserves runtime data and configuration while
rebuilding the application from the selected Git ref.

Before rollout, back up `/opt/roon-ai-bridge/data` and the LXC `.env`. A rollback
should select the previous validated Git tag and rebuild through the documented
LXC update procedure; do not copy database files from a newer running process.

See [Update Existing LXC](update-lxc.md), [MCP v2 Architecture](mcp-v2-architecture.md)
and [v0.17.0 Validation](v0.17.0-validation.md).
