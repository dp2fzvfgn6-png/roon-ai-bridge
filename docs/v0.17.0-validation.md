## Included scope

- Persistent playlist recording identity and safe playback reconstruction.
- Candidate ranking that prefers clean studio recordings and reports ambiguity.
- Portal redesign, active-zone controls and authenticated artwork.
- Canonical MCP v2 facade with 34 exposed tools.
- Player, media explorer and library widgets under the v10 resource namespace.
- ChatGPT OAuth and generic MCP connection management.
- Hardened Roon SDK boundaries, state cache and transport-related services.
- Standalone LXC update watcher and build-aware update reporting.

Detailed behavior is described in [release notes](v0.17.0-release-notes.md),
[MCP v2 validation](mcp-v2-validation.md), [widget v2 validation](widget-v2-validation.md),

## Automated validation

Run from the repository root:

```powershell
pnpm run test
pnpm run build
git diff --check
git diff --stat
git status --short
```

Expected requirements:

- TypeScript compilation succeeds.
- The complete Node test suite passes with zero failures.
- Version assertions report `0.17.0`.
- Diff checks contain no whitespace errors.
- Only intentional release files are changed before integration.

Results on 2026-07-13:

- `pnpm run test`: passed, 105 tests, 105 passed, 0 failed. The command includes
  a TypeScript compilation before the Node test suite.
- `pnpm run build`: passed (`tsc`).
- Version assertions reported `0.17.0`.
- Final diff and worktree checks are recorded before integration.

## Required live validation before tagging

Deployment requires a separate explicit request. Once authorized:

1. Deploy the exact release commit to `/opt/roon-ai-bridge` in the LXC.
2. Confirm the deployed Git commit matches `origin/main`.
3. Confirm `package.json`, `/version` and MCP initialization report `0.17.0`.
4. Confirm `/roon/status` reports core, transport and browse ready.
5. Confirm `tools/list` exposes the 34 MCP v2 tools, their schemas,
   annotations and the v10 widget URIs.
6. Read the three widget resources through MCP.
7. Exercise changed behavior through the public MCP tool path used by ChatGPT.
8. Use Despacho for audible checks, preserving volume and reporting the final
   playback, queue and grouping state.
9. Refresh the ChatGPT app and start a new conversation before host-level
   widget/tool-selection validation.

## Tagging gate

Create and push `v0.17.0` only after every required automated and live check
passes. Do not tag a locally validated but undeployed release candidate.
