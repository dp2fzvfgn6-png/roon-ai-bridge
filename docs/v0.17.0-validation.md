## Included scope

- Persistent playlist recording identity and safe playback reconstruction.
- Candidate ranking that prefers clean studio recordings and reports ambiguity.
- Portal redesign, active-zone controls and authenticated artwork.
- Canonical MCP v2 facade with 34 exposed tools.
- Player, media explorer and library widgets under the v11 resource namespace.
- ChatGPT OAuth and generic MCP connection management.
- Hardened Roon SDK boundaries, state cache and transport-related services.
- Standalone LXC update watcher and build-aware update reporting.

## Live validation

The first deployment of `2bca75a` confirmed version, Roon readiness, the MCP
catalog and real search behavior, but the pre-tag resource inspection found
that widget v10 did not consume `ui/notifications/tool-input` or
`window.openai.toolInput`. The tag was stopped.

The widget bridge was corrected, covered by automated tests and cache-busted
to v11. Commit `1a64c627269835f6b80461e700ec622da3afdf20` was deployed and
validated again through the same MCP endpoint used by ChatGPT.

Results:

- Deployed Git commit matched `origin/main` and the container build identifier.
- Host package, container package, `/version` and MCP initialization reported
  `0.17.0`.
- The container was running with no startup error/fatal log lines.
- `/roon/status` reported Roon Server connected; transport, browse and image
  ready; 4 zones and 4 outputs.
- `tools/list` exposed 34 tools with input/output schemas, `Use this when...`
  descriptions, correct visibility and no legacy aliases.
- The three render tools referenced the v11 player, media explorer and library
  resources; `roon_ui_navigate` remained app-only.
- All three resources used `text/html;profile=mcp-app` and contained the MCP
  Apps tool-input/tool-result bridge plus ChatGPT `toolInput`, `toolOutput`,
  `callTool` and `setWidgetState` compatibility globals.
- `roon_get_state` resolved Despacho by name and returned verified state.
- `roon_search_media("Max Cooper Repetition")` returned four real candidates
  with `Repetition` by Max Cooper first and a recommended result identifier.
- `roon_get_queue` and `roon_open_player` both completed through MCP; the player
  returned four zones and the Despacho queue preview.
- No playback, queue, grouping or volume mutation was performed. Despacho kept
  playing at volume 30 and its queue remained unchanged.

After recording this evidence, the final documentation-only release commit is
re-deployed and smoke-checked for commit/version parity before tagging.

## Tagging gate

Every required automated and live check passed. The release is approved for
the annotated `v0.17.0` tag.
