# v0.20.0 Beta 3 Explainable Matching Validation

## Scope

Beta 3 remains read-only. It explains recording failures and audits the
contradiction found in beta 2 between legacy `exact` metadata and identity V2.

It adds:

- `cache_hit`, cache layer, elapsed time and provider request counts;
- search attempts and up to twelve rejected candidates with reason codes;
- complete MusicBrainz artist credits, MBIDs and join phrases;
- strict artist equality instead of partial-name containment;
- direct verification of a stored recording MBID before running a new search;
- `stored_metadata_audit` for exact durations without release-track evidence.

`Jimi Hendrix` and `The Jimi Hendrix Experience` remain separate credits. A
stored Purple Haze recording MBID may still be verified when its title/version
match; the distinct MusicBrainz artist credit is returned as a warning and does
not replace playlist intent.

## Rejection and audit reasons

Candidate reasons:

- `invalid_recording_shape`
- `title_mismatch`
- `artist_mismatch`
- `release_mismatch`
- `isrc_mismatch`
- `duration_mismatch`
- `variant_mismatch`

Stored metadata issues:

- `stored_exact_duration_lacks_release_track_provenance`
- `stored_exact_conflicts_with_shadow_recording_status`
- `stored_duration_differs_from_verified_release_track`

## Validation

After deployment:

1. Analyze Purple Haze, Born To Be Wild and Won't Get Fooled Again.
2. Inspect `provider_result.trace`, `release_result.provider_trace` and
   `stored_metadata_audit`.
3. Confirm the first uncached call has provider requests; repeat it and confirm
   `cache_hit=true`, `cache_layer=memory`, `provider_requests=0`.
4. Restart RoonIA and confirm the next hit uses `cache_layer=persistent`.
5. Purple Haze must either verify stored recording MBID
   `107af95b-776e-40af-a677-328ce4a72f16` or state the exact rejection reason.
6. A verified MBID with a different group/person credit must preserve both
   identities and add `artist_credit_differs_from_intent`.
7. Any stored duration sourced only as `musicbrainz` must remain flagged until
   an exact release track independently verifies it.
8. Confirm the playlist remains unchanged.

No write path may consume these results until the real beta report contains a
positive exact release, a release-group candidate and no unexplained legacy
`exact` conflicts.
