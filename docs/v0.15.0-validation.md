# Roon AI Bridge v0.15.0 validation

Date: 2026-07-09

## Scope

Lot 6 adds reusable widget contracts for ChatGPT Apps and the RoonIA portal:

- Now playing widget contract and actions.
- Virtual playlists list/detail widget contracts and actions.
- Playlist-created UI hint for playlist mutation tools.
- Media search widget contract and actions.
- Roon image URL endpoint for artwork without base64 payloads.
- Portal Widgets view backed by the same `/widgets/*` contracts.

Home Assistant, Extension Manager and advanced snapshots are intentionally out of scope.

## ChatGPT widget capability research

OpenAI Apps SDK documentation supports Strategy A for real interactive widgets:

- Apps can provide custom UI components in iframes that render MCP tool `structuredContent`.
- Components receive `ui/notifications/tool-input` and `ui/notifications/tool-result`.
- Components can call MCP tools directly with `tools/call` / `window.openai.callTool`.
- Components can keep local state with `window.openai.widgetState` / `setWidgetState`.
- Render tools should advertise `_meta.ui.resourceUri` and ChatGPT compatibility `_meta["openai/outputTemplate"]`.
- The current RoonIA app resource uses `text/html;profile=mcp-app` and supports the MCP Apps bridge plus `window.openai` compatibility globals.

Implemented strategy: Strategy A, with clean reusable JSON contracts first and a single cache-busted widget resource `ui://roon-ai-bridge/control-v7/{tool}.html`.

Primary references:

- https://developers.openai.com/apps-sdk/build/chatgpt-ui
- https://developers.openai.com/apps-sdk/reference

## Contract notes

Artwork is exposed as both `image_key` and `image_url`. Payloads do not include base64 by default.

Navigation is explicit in each widget payload:

```json
{
  "navigation": {
    "current_view": "playlist_detail",
    "can_go_back": true,
    "back_stack": [{ "view": "list" }]
  }
}
```

Actions return an `ok`, `action`, `message`, optional `result`, and `refresh` or nested `widget` payload.

## Known limitations

- Album track listing and artist popular tracks depend on deeper Roon browse detail than the cached `result_id` currently stores. The widget returns clear warnings and `bio:null` rather than inventing metadata.
- Widget playback actions are real actions. Live validation should avoid audible playback unless explicitly approved.
- ChatGPT users must refresh the app and start a new conversation to load changed tool metadata and `ui://` resources.
