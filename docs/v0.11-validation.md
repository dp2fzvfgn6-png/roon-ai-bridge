# v0.11 Validation

v0.11 adds the RoonIA Control administration portal on port `3001` and
revocable, role-based API keys.

## Automated Validation

```bash
pnpm run test
pnpm run build
node --check portal/app.js
```

Local validation covers:

- Managed API secrets are stored only as SHA-256 hashes.
- List responses never include the complete secret.
- Revoked keys stop authenticating.
- Read-only keys cannot enter the administration portal.
- The environment bootstrap token and managed admin keys can enter.
- Static assets and `/api/health` load without authentication but every
  administration route is protected.
- Existing Roon, OAuth, MCP, playlist, grouping and transfer tests remain
  green.

## Browser Validation

Validated locally against the real Express portal:

- Login with the administrative bootstrap token.
- Dashboard disconnected-state rendering.
- Desktop navigation and responsive mobile navigation.
- API-key creation form and one-time secret dialog.
- API-key inventory refresh.
- No browser console errors during the tested flow.

## Live Validation

Pending deployment checks:

- Docker exposes the portal through host networking on `10.0.60.38:3001`.
- Login succeeds with the configured bootstrap credential.
- Dashboard reports the real Roon Core and zones.
- Playback, volume, queue and native transfer controls return verified live
  state.
- Existing SQLite playlists are visible and editable without data loss.
- A managed admin key can log in and a revoked key cannot.
- API/MCP behavior on port `3000` remains compatible.
