# v0.9.1 Validation

v0.9.1 adds native Roon playback transfer and improves MCP tool selection.

## Automated Tests

```bash
pnpm run test
```

Validated locally:

- The bridge calls Roon `transfer_zone` exactly once with the source and target zones.
- Source and target must be different.
- The transfer result identifies both zones and reports `queue_and_playback`.
- MCP server instructions tell ChatGPT not to search or rebuild the queue.

## Live Validation

Use the deployed Despacho and Cocina zone IDs:

```bash
curl -X POST http://localhost:3000/roon/zones/transfer \
  -H "Authorization: Bearer <API_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"source_zone_id":"<DESPACHO_ZONE_ID>","target_zone_id":"<COCINA_ZONE_ID>"}'
```

Expected result:

- Current playback and the complete queue move from Despacho to Cocina.
- No media search or queue reconstruction occurs.
- `roon_transfer_playback` is visible to both the model and the app.
