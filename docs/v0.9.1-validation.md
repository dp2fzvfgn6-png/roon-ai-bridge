# v0.9.1 Validation

v0.9.1 adds native Roon playback transfer and improves MCP tool selection.

## Automated Tests

```bash
pnpm run test
```

Validated locally:

- The bridge calls Roon `transfer_zone` exactly once with the source and target zones.
- Source and target must be different.
- The transfer result identifies both zones and reports `queue_and_playback`.
- MCP server instructions tell ChatGPT not to search or rebuild the queue.

## Live Validation

Validated against the deployed Roon Core:

```bash
curl -X POST http://localhost:3000/roon/zones/transfer \
  -H "Authorization: Bearer <API_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"source_zone_id":"16017884276cc06e0e4c92d966765d65ec86","target_zone_id":"1601f17521ff85962b1f008000a96f1c3455"}'
```

Validated result:

- Cocina changed from `Rock Stone` to the same `World on Fire` playback
  position held by Despacho.
- The first five destination queue entries exactly matched the source queue,
  in the same order.
- The paused playback state was preserved.
- No media search or queue reconstruction occurs.
- `roon_transfer_playback` is visible to both the model and the app.
- MCP initialization reports version `0.9.1` and instructs ChatGPT to call the
  transfer tool exactly once instead of rebuilding the queue.
