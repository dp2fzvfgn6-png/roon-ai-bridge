# v0.8 Validation

v0.8.1 validates the ChatGPT app OAuth preparation layer.

## Local Build

```bash
pnpm run build
```

Expected:

```text
tsc
```

## Public Checks

These checks do not need the token:

```bash
curl https://roonia.ipchome.com/health
curl https://roonia.ipchome.com/privacy
```

Expected:

- `/health` returns the service health JSON.
- `/privacy` returns a plain text privacy notice.

## MCP Auth Check

Without a token:

```bash
curl -i https://roonia.ipchome.com/mcp
```

Expected:

```text
401 Unauthorized
```

## OAuth Discovery

```bash
curl https://roonia.ipchome.com/.well-known/oauth-protected-resource
curl https://roonia.ipchome.com/.well-known/oauth-authorization-server
```

Expected:

- Protected resource metadata points to `https://roonia.ipchome.com/mcp`.
- Authorization server metadata includes `/oauth/authorize`, `/oauth/token` and `/oauth/register`.

## Dynamic Client Registration

```bash
curl -X POST https://roonia.ipchome.com/oauth/register \
  -H "Content-Type: application/json" \
  -d '{"client_name":"ChatGPT","redirect_uris":["https://chatgpt.com/connector/oauth/test"]}'
```

Expected:

- `201 Created`
- Response includes `client_id`
- `token_endpoint_auth_method` is `none`

With the token, use an MCP client or MCP Inspector against:

```text
https://roonia.ipchome.com/mcp
```

Expected:

- The server initializes as `roon-ai-bridge`.
- The tool list includes `roon_status`, `roon_list_zones`, `roon_control_playback`, `roon_change_volume`, `roon_search`, `roon_play_by_query`, queue tools and virtual playlist tools.
- The server exposes the widget resource `ui://roon-ai-bridge/control-v1.html`.

## Notes

v0.8.1 is not a Custom GPT Action integration and does not use an OpenAPI schema. It is the first remote MCP/Apps SDK preparation step for a ChatGPT app with OAuth.
