# RoonIA v0.17.3 Release Notes

Date: 2026-07-20
Release channel: stable (`main`)
Source channel: beta

## Summary

v0.17.3 promotes the tested beta line that followed v0.17.2. It adds expiring
working playlists for contextual listening, completes deep reads of native
Roon playlists, expands the read-only ChatGPT widget set and refines persistent
portal playback controls.

## User-visible changes

- Temporary playlists keep activity, mood and occasion requests outside the
  saved collection. They can replace or extend a queue, expire automatically
  and be promoted in place when the user wants to keep them.
- System settings expose a configurable 1-365 day temporary-playlist lifetime;
  Debug mode adds a dedicated manager for inspection and maintenance.
- Native Roon catalog playlists can be read in ordered pages, including lists
  longer than 100 tracks, so they can be compared with RoonIA playlists.
- The persistent portal player adds compact per-zone and grouped-output volume
  controls while preserving native volume steps and configured safety limits.
- Queue rows now expose normalized title, artist and album metadata.
- The MCP Apps surface grows from three to six focused read-only widgets with
  saved-playlist library, bounded queue and all-zone views.

## MCP and widget compatibility

The canonical MCP v2 intent boundary remains in place. Three new model-visible
render tools expose playlist-library, queue and zone-panel views. All six
read-only resources use the cache-busted `ui://roon-ai-bridge/v19/` namespace.

After deployment, refresh the ChatGPT app and start a new conversation so it
reloads tool metadata and widget resources.

## Upgrade and rollback

Stable installations follow `main`; installations with beta updates enabled
follow `beta`. The LXC updater preserves runtime data and configuration while
rebuilding from the selected Git ref. Existing databases are migrated in place
for the temporary-playlist lifecycle fields.

Before rollout, back up `/opt/roon-ai-bridge/data` and the LXC `.env`. Rollback
should select the previous validated tag and use the documented LXC update
procedure.
