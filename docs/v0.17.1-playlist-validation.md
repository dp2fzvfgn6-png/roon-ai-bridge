# v0.17.1 Playlist MCP Validation

Status: local implementation and automated validation complete; deployment and
live MCP validation are pending an explicit deployment request.

## Behavior

- `roon_save_playlist` resolves text-only tracks and accepts playable track
  `result_id` references from `roon_search_media` as exact selections.
- `roon_edit_playlist_tracks` publishes explicit add, update, remove, reorder
  and replace schemas. Additions and replacements use the resolving service
  paths instead of storing unassociated text rows.
- Updating a track with `changes.result_id` replaces an incorrect association
  with the exact user-selected playable recording.
- `roon_resolve_playlist` supports unresolved, selected-track and full-playlist
  repair scopes.
- `roon_set_playlist_cover` exposes the existing validated JPEG, PNG and WebP
  custom-cover storage to the MCP v2 catalog used by ChatGPT.

## Automated evidence

Focused tests cover text-track resolution, exact search-result identity,
selected repair, exact manual reassociation, custom cover forwarding and the
34-tool HTTP manifest. The playlist service regression suite also verifies
custom cover persistence and fresh playback reconstruction.

Run the final validation with:

```powershell
pnpm run test
pnpm run build
git diff --check
```

After a future deployment, refresh the ChatGPT app and start a new conversation
so it reloads the updated tool names, descriptions and schemas.
