# RoonIA v0.17.1 Release Notes

Date: 2026-07-16
Release channel: stable (`main`)
Source channel: beta

## Summary

v0.17.1 promotes the tested beta line that followed v0.17.0. It improves
catalog discovery, playlist recording selection, embedded widget artwork and
the portal music experience without changing the established MCP v2 boundary.

## User-visible changes

- Search follows Roon's preferred result while exposing stable groups for
  artists, albums, EPs, singles, tracks and playlists.
- Artist views provide richer discographies and album views recover complete
  ordered track lists, including multi-disc releases.
- Automatic playlist association avoids unintended live, remix and cover
  variants, preserves ambiguous results and prefers TIDAL or the best known
  quality only after identifying an equivalent recording.
- Playlists support normalized custom artwork, paginated widget views and
  recent-playback ordering. The portal presents library browsing as `Mi Música`.
- Widget artwork uses signed URLs through the public MCP route and playlist
  views fall back to Roon artwork when no custom cover exists.

## MCP and widget compatibility

The MCP facade still exposes 34 tools: 31 canonical intents and three
model-visible widget entry points. The current resources use the cache-busted
`ui://roon-ai-bridge/v17/` namespace.

Playlist mutations report a `resolution_summary` and never claim verification
while a track is ambiguous, missing or failed. Temporary Roon references are
reconstructed with the same strict recording policy before playback.

After deployment, refresh the ChatGPT app and start a new conversation so it
reloads tool metadata and widget resources.

## Upgrade and rollback

Stable installations follow `main`; installations with beta updates enabled
follow `beta`. The LXC updater preserves runtime data and configuration while
rebuilding from the selected Git ref. Existing databases are migrated in place
with the playlist recent-playback field.

Before rollout, back up `/opt/roon-ai-bridge/data` and the LXC `.env`. Rollback
should select the previous validated tag and use the documented LXC update
procedure.
