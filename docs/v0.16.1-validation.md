# RoonIA v0.16.1 Validation

Date: 2026-07-09
Status: prepared locally, not deployed, not tagged.

## Scope

The beta now also applies the persistent RoonIA identity model to virtual
playlists:

- keep `track_id` as the permanent local identity;
- persist a complete recording snapshot and versioned identity fingerprint;
- treat `roon_item_key` and `result_id` as temporary observations only;
- reconstruct and score a fresh Roon reference before every playback action;
- leave the current queue unchanged when the first `play_now` identity is
  missing or ambiguous;
- keep resolver `status` and `confidence` coherent;
- prefer clean studio/original search results over alternate versions;
- classify and penalize remix, edit, remaster, live, binaural, 3D and interpretation versions;
- expose `is_best_match`, `selection_required`, `ambiguity_reason` and version penalties in search results;
- expose known unavailable outputs through `roon_list_outputs(include_unavailable)`;
- make zone preset dry-runs warn clearly for unavailable outputs.

## Search Examples Covered By Tests

`roon_search_media("Max Cooper Repetition")` with candidates:

1. `Repetition` - Max Cooper
2. `Repetition (Edit)` - Max Cooper
3. `Repetition 3D (Binaural Version - Headphones Only)` - Max Cooper
4. `Repetition (Josh Wink Interpretation)` - Max Cooper

Expected and validated:

- `Repetition` is first;
- `recommended_result_id` points to the clean result;
- `is_best_match` is true only for the clean result;
- `Repetition 3D...` is classified as `alternate`, not `cover`;
- binaural/3D and interpretation penalties are present.

`roon_search_media("Massive Attack Angel")` with candidates:

1. `Angel` - Massive Attack
2. `Angel (Remastered 2006)` - Massive Attack
3. `Angel (Blur Remix)` - Massive Attack

Expected and validated:

- `Angel` is first;
- `ambiguous` is false;
- `selection_required` is false;
- `Angel (Blur Remix)` is `remix`;
- `Angel (Remastered 2006)` is `remaster`.

## Virtual Playlist Examples Covered By Tests

Creating a virtual playlist with:

```json
{
  "tracks": [
    {
      "query": "Max Cooper Repetition",
      "title": "Repetition",
      "artist": "Max Cooper",
      "album": "Yearning for the Infinite"
    },
    {
      "query": "Massive Attack Angel",
      "title": "Angel",
      "artist": "Massive Attack",
      "album": "Mezzanine"
    }
  ]
}
```

Expected and validated:

```text
resolved: 2
unresolved: 0
ambiguous: 0
```

Each track stores:

- permanent `track_id`;
- `identity.fingerprint`, canonical query and recording metadata;
- a complete `resolution.selected_candidate` snapshot;
- the last observed `roon_item_key` only for diagnostics;
- `resolution.roon_item_key_persistent = false`;
- `roon_binding.reusable = false` and `roon_binding.state = stale` after the
  Browse session ends;
- `resolution.status = resolved`;
- coherent confidence.

Validation reports `resolved`, `manual`, `stale`, `ambiguous`, `missing` and
`error` separately. Deduplication uses the RoonIA identity fingerprint and
normalized recording metadata by default; equal `roon_item_key` values do not
make two tracks duplicates.

## Manual Selection

Existing manual-selection coverage still passes:

- `roon_set_virtual_playlist_track_match` persists the selected recording
  snapshot and only caches its temporary Roon reference;
- manual selection preserves user metadata;
- add-from-search-result still stores manual resolution metadata.

## Playback Reconstruction

Automated coverage verifies that production playlist playback:

- searches from the stored canonical identity;
- selects a fresh temporary `result_id` and re-resolves it in the action Browse
  session;
- never invokes a persisted `roon_item_key`;
- uses `replace_queue` for the first item and `append` for the remainder;
- explicitly starts playback after rebuilding the queue;
- aborts before changing the queue when the first match is ambiguous.

## Outputs

Automated coverage validates:

- visible outputs return `currently_available: true`;
- known unavailable outputs return `currently_available: false` when `include_unavailable: true`;
- unavailable outputs expose `last_seen`, `last_known_zone_id` and `last_known_volume_type`;
- presets referencing unavailable outputs return dry-run warnings without internal errors.

## Regression Coverage

The full suite still covers zone presets, grouping, queue, widgets, safe volume policy, `roon_evaluate_volume_policy`/volume control behavior and MCP tool manifests.

## Metadata Enrichment Limitations

The search layer continues to preserve metadata directly exposed by Roon Browse items. When Roon does not expose source, quality, library status, album or album artist, the result keeps `null`/`unknown` and emits explicit warnings rather than inventing values.
